## Context

Phase 5 established the Resend email pattern with lazy initialization. The sushi order form demonstrates the client-side validation and API submission flow. The contact page already has a complete form UI but uses a simulated submission. The bookings page needs to be created from scratch following the reference site's content.

**Key decisions:**
- WhatsApp is the **primary** booking channel (no booking form needed)
- Contact form gets Resend integration as backup inquiry method
- Bookings page is **informational** with prominent WhatsApp CTA
- No complex booking form required (enquiry-based, not self-service booking)

## Tasks

### 1. Create bookings data file
**File:** `src/data/bookingsData.ts`

Create data file with:
- Event types array (Celebrations, Memorial Gatherings, Business Meetings)
- What's included items array
- Booking process steps array
- Venue info (capacity: 30 guests)

### 2. Create BookingsPage
**File:** `src/app/bookings/page.tsx`

Sections (following reference layout):
1. PageHeader with "Private Bookings" title, private-room.webp background
2. WaveSeparator
3. Our Venue section — 2-column grid with text + privateroom2.webp image
4. Catering section — centered text + catering.webp image
5. Event Types section — 3-column grid with icons (Cake, BookOpen, Briefcase)
6. Booking Information section — 2-column grid:
   - Left: "What's Included" with checkmarks
   - Right: "Booking Process" numbered steps
7. Ready to Book CTA section — prominent WhatsApp button + phone/email links

Use existing components: PageHeader, SectionHeader, WaveSeparator

Add metadata export for SEO.

### 3. Create contact validation file
**File:** `src/lib/contactValidation.ts`

Create validation functions:
- `validateEmail(email: string): boolean` — email format check
- `validateContactForm(data: ContactFormData): string[]` — returns array of error messages
- Interface `ContactFormData` with name, email, subject, message fields

### 4. Create contact API route
**File:** `src/app/api/contact/route.ts`

Following sushi-order pattern:
- Lazy Resend initialization
- POST handler with server-side validation
- Send email to `info@sweetlifecafe.co.uk` with:
  - From: `Sweet Life Website <noreply@sweetlifecafe.co.uk>` (or Resend default)
  - Subject: `[Website Inquiry] {subject}`
  - HTML template with sender info + message
- Return success/error JSON response
- Graceful handling when RESEND_API_KEY not configured

### 5. Wire up Contact page form
**File:** `src/app/contact/page.tsx` (modify)

Changes:
- Add FormData interface
- Add client-side validation using contactValidation.ts
- Replace simulated submission with actual fetch to `/api/contact`
- Add proper error display (field-level or summary)
- Keep existing success message pattern
- Add loading state during submission

### 6. Add WhatsApp booking CTA component
**File:** `src/components/ui/WhatsAppBookingCTA.tsx`

Create prominent WhatsApp CTA component for bookings page:
- Large WhatsApp icon
- "Book via WhatsApp" button with primary styling
- Pre-filled message: "Hi! I'd like to enquire about booking your private event space."
- Optional secondary text for phone/email alternatives
